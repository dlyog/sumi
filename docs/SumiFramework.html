SUMI FRAMEWORK

Add a voice-first AI Learning Companion to any screen.

Sumi combines an interruptible browser voice SDK, a bounded JSON action registry, prepared explanations, and three replaceable AI provider adapters.

Install 1StopQuantum and Sumi

On a newly cloned Apple-silicon Mac, the root installer bootstraps Homebrew when necessary, installs Python, Node, and PostgreSQL, reuses or provisions the database, installs dependencies, and builds the integrated application.

git clone <repository-url> sumi
cd sumi
./setup.sh
ALLOW_LLM_UNAVAILABLE=1 ./manage.sh start
open http://localhost:8080

Sumi is already integrated into 1StopQuantum. The remaining sections describe how to copy its browser assets and scaffold a bounded adapter for another learning platform.

Bring three APIs

Speech-to-textLocal MLX Whisper accepts recorded or streaming learner audio on port 5152 and returns a transcript.
LLMClassify only registered actions and answer unmatched questions. Text Gemma is the local reference adapter.
Text-to-speechLocal Kokoro returns playable WAV audio from the same port-5152 service.
./scripts/setup-local-voice.sh
./manage.sh start
curl -fsS http://127.0.0.1:5152/health

The local API provides POST /api/transcribe, WS /api/duplex, and POST /api/speak. LLM reasoning uses a separate OpenAI-compatible /v1 server configured in .env. See the complete local AI setup for Ollama, MLX-LM, llama.cpp, and verification commands.

Scaffold a screen

Installed as a package, run sumi-framework init. Inside this repository, use the npm wrapper:

npm run sumi-framework -- init \
  --screen lesson-lab \
  --title "Lesson Lab" \
  --out ./public/sumi

npm run sumi-framework -- validate ./public/sumi/lesson-lab.registry.json

The CLI creates a provider configuration, a versioned screen registry, and an adapter. Endpoint values are runtime configuration; API secrets must remain on the server.

CLI reference

  • sumi-framework init --screen <id> --title <title> --out <directory> creates the provider config, registry, and adapter.
  • sumi-framework validate <registry.json> validates the version, screens, allowed actions, unique IDs, and required provider contract.
  • Inside this repository, prefix either command with npm run sumi-framework --.

Registry contract

Each page declares stable action IDs, user-language aliases, screen terms, detailed explanations, control IDs, and optional prepared-audio IDs. The LLM never returns selectors or executable code. The application validates the action and owns every screen mutation.

{
  "screen_id": "lesson-lab",
  "actions": [{ "id": "explain_screen", "aliases": ["explain this screen"] }],
  "terms": [{
    "id": "result_chart",
    "aliases": ["result chart"],
    "control_id": "resultChart",
    "description": "A detailed reviewed explanation.",
    "audio_id": "result_chart"
  }]
}

UI tools with verified results

For actions such as changing a learning level, opening a lesson, loading a circuit template, or reading a measurement, register a typed handler with SumiUIActions. The assistant receives only the action IDs allowed by the active screen; each handler returns a short spoken result and optional data read-back.

window.SumiUIActions.register("set_learning_level", ({ level }) => {
  learningStore.setLevel(level);
  return { text: `Learning level is now ${level}.`, data: { level } };
}, { properties: { level: { type: "string", enum: ["High school", "Undergraduate", "Master's"] } }, required: ["level"] });
window.SumiUIActions.allow(screen.allowed_actions);
await window.SumiUIActions.execute("set_learning_level", { level: "High school" });

Conversation behavior

Fast, interruptible, and honest. Stream microphone audio, suppress startup self-echo, use VAD for barge-in, prefer prepared audio, and fall back to LLM plus real-time TTS. This is an STT → LLM → TTS pipeline with a full-duplex feel, not native speech-to-speech.

Production checklist

  • Request microphone access only after a clear learner action.
  • Enable echo cancellation, noise suppression, VAD endpointing, and immediate playback cancellation.
  • Keep provider keys server-side and allow only trusted application origins.
  • Version registries, test every action handler, and provide keyboard and visible transcript alternatives.
  • Say when a result is simulated and never let an LLM claim an action the application did not verify.
